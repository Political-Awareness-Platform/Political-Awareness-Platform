# Security Policy

## Supported Versions

We support security updates for the current major version of the Political Awareness Platform.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it to us privately before disclosing it publicly.

### How to Report

- Email us at: politicalawarenessplatform@gmail.com
- Include "Security Vulnerability" in the subject line
- Provide as much detail as possible about the vulnerability

### What to Expect

- We will respond within 48 hours to acknowledge receipt
- We will provide a detailed response within 7 days
- We will work with you to understand and validate the report
- We will provide a timeline for fixing the vulnerability
- We will credit you in our security advisories unless you prefer to remain anonymous

### What to Include in Your Report

- Type of vulnerability (e.g., XSS, SQL injection, etc.)
- Steps to reproduce the vulnerability
- Potential impact of the vulnerability
- Any screenshots or proof-of-concept code (if available)

## Security Best Practices

We follow these security best practices:

- Regular dependency updates
- Code reviews for all changes
- Security testing before releases
- Using HTTPS everywhere
- Proper input validation and sanitization
- Secure authentication and authorization

## Security Advisories

Security advisories will be published on our [GitHub Security Advisories](https://github.com/Political-Awareness-Platform/Political-Awareness-Platform/security/advisories) page.

Thank you for helping keep the Political Awareness Platform safe!
